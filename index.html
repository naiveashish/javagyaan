Variables aur Data Types â€“ Bhidu, pocket mein kya rakhega?

Java mein variables ka matlab hota hai tera "data storage". Jaise tu apne pockets mein paise, phone, aur gum rakhta hai, waise hi variables mein alag-alag type ka data rakha jata hai. Ab sun:

1. int â€“ Integer (Whole Numbers)

Yeh tere normal numbers ke liye hota hai. Example: int age = 25;. Toh jab tu apni umar ya marks store karega, toh "int" ka use karega.

"Abey yeh int ka data bada solid hai, zero point point ka tension nahi!"

2. float â€“ Floating Point Numbers (Chhote Chhote Decimal Values)

Yeh tere decimal ke liye hota hai. Jaise ki agar tu kisi item ka price store kar raha hai: float price = 99.99f;. Bhai, decimal ka last mein 'f' lagana mandatory hai.

"Bhai, price toh fix hai, 'f' laga na bhoolna!"

3. double â€“ Bada Decimal Number

Double apna bada bhaisaab hai, bade decimals ko store karta hai. Jaise: double distance = 12345.6789;. Jab zyada precision chahiye, tab double use karna.

"Arey bhai! Double ka double dhamaka! Precision ekdum on point!"

4. char â€“ Character

Yeh sirf ek akshar ke liye hota hai, jaise tu apna initial ya grade store karega: char grade = 'A';. Dhyaan rahe, char sirf single letter ko store karega!

"Single hai toh char mein hi fit hoga!"

5. boolean â€“ True ya False ka Khel

Isme sirf do value hoti hai â€“ ya toh sach ya jhooth, ya toh true ya false. Example: boolean isCool = true;. Jab koi condition check karni ho, tab boolean ka use hota hai.

"Tera dost cool hai ya nahi? boolean mein check kar!"

6. String â€“ Chhota Ya Bada Full Sentence

Yeh apna favorite hai, poora sentence ya naam store karta hai. Example: String naam = "Bhaijaan";. Jab tu text ya naam store karega, toh String ka use hoga.

"String ka bada scene hai, poora naam aur full dialogue store karta hai!"

Loop ka Chakkar â€“ Kaam Repeat Karna Full Easy!

Life mein jaise tu baar baar ek hi kaam karta hai, waise programming mein bhi loops se repeat karte hain! Loop ka funda hai, jab tak condition true hai, tab tak kaam chalne de.

Jab tu koch bhi cheez baar baar karna chahta hai, for loop kaam aayega. Jaise, tu 5 baar "hello" bolna chahta hai, toh for loop me kaam pura ho jaayega!

"Abe chal chal! Loop mein ghus ja, kaam khatam hone tak niklega nahi!"

OOPs Concepts â€“ Bhai, sab log ke baap yeh!

OOPs ka matlab â€“ Encapsulation, Inheritance, Polymorphism, Abstraction. Yeh sab samjhe bina tu Java ka asli game samajh nahi sakta.

Encapsulation â€“ Data Chhupane Ka Tareeka

Encapsulation ka matlab hai wrapping data and methods into a single unit, jisse bahar waale tere data ko directly access na kar sake. Public getters aur setters ke through hi private data ko modify kar paayenge. Example dekh:

class Car {
    private String brand;
    public void setBrand(String brand) { this.brand = brand; }
    public String getBrand() { return brand; }
}

"Beta, itna encapsulation karo ki duniya confuse ho jaye!"

Inheritance â€“ Baap Ka Property Beta Ko Milega

Inheritance ka funda hai, ek class doosri class ke properties aur methods ko inherit kar sakti hai. Jaise tu apne baap ka property apne naam pe likhwa sakta hai. Example:

class Parent {
    public void show() {
        System.out.println("Baap ka property!");
    }
}
class Child extends Parent {
    public void show() {
        super.show();
        System.out.println("Aur ab beta ka bhi!");
    }
}

Yaha `super` keyword use hota hai baap ka method call karne ke liye, aur uske baad beta apna kaam karega.

"Baap ka property sabko nahi milta, inherit karke le bhidu!"

Polymorphism â€“ Ek Kaam Alag Tareeke Se

Polymorphism ka matlab hai ek cheez alag-alag tareeke se karna. Java mein do types ke polymorphism hote hain â€“ compile-time aur runtime. Example:

class MathFun {
    public int add(int a, int b) { return a + b; }
    public float add(float a, float b) { return a + b; }
}

Method overloading ke through, hum same function ko alag input ke saath use kar sakte hain. Runtime polymorphism ke liye overriding ka use hota hai.

"Ek kaam 2 tareeke se kar, usko bolte hai polymorphism!"

Strings â€“ Java Ke Dialogue Writer

Strings matlab text ya dialogue, Java mein Strings are immutable, yeh ek baar ban jaye toh change nahi hota, naye address pe naya object banta hai. Example:

String s1 = "Bhaijaan";
s1 = "Sallu";
System.out.println(s1); // Output: Sallu

Jab tu "Bhaijaan" ko "Sallu" banaya, toh naya object ban gaya. Purana string waisa ka waisa hi rahega, naya address pe "Sallu" store hoga.

"String se zyada to dialogues change kar leta tu, samjha?"

Collections â€“ Data Ko Sametne Ka Tareeka

Collections ka kaam hai, ek jagah pe data ko store karna. Java Collections framework mein 3 main interfaces hain â€“ List, Set, aur Map. Sabka alag purpose hai.

List â€“ Ekdum Line Mein

List ek ordered collection hai, jisme duplicate values allowed hain. Example:

List names = new ArrayList<>();
names.add("Bhaijaan");
names.add("Sallu");
names.add("Bhaijaan");
System.out.println(names); // Output: [Bhaijaan, Sallu, Bhaijaan]

Yeh ekdum line mein store hota hai, aur index ke through access hota hai.

"List mein tu duplicate ho sakta hai, bas line mein lag ja!"
